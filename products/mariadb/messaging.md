# Percona for MariaDB: Messaging

## Percona for MariaDB {#percona-for-mariadb}

For organizations running MariaDB who need reliable production operations, a path off proprietary licensing, or a controlled move to Percona's MySQL stack, Percona provides **Expert Support** for **MariaDB Community**, **Expert Consulting and Services** for complex MariaDB estates (including versions outside Community Support scope), and migration guidance to **Percona Server for MySQL** and the broader Percona software for MySQL suite when that is the right technical direction.

Percona does not ship MariaDB database software. We support **MariaDB Community** in production and bring the same upstream-level engineering depth we apply across MySQL, PostgreSQL, MongoDB, and Valkey. When MariaDB no longer fits performance, operational, or licensing requirements, Percona helps teams plan and execute a **controlled migration** to Percona Server for MySQL with validated testing and a cutover plan built for their environment.

**Messaging guardrail (MariaDB scope):** Percona **Expert Support** covers **MariaDB Community versions only**. Do not imply support for MariaDB Enterprise editions. Custom-scoped consulting may apply to other MariaDB versions and complex environments. Ref: [MariaDB support](https://www.percona.com/services/support/mariadb-support).

### How customers engage

1. **Stay on MariaDB Community:** 24×7 **Expert Support** for production incidents, configuration, performance, high availability, backup and recovery, and upgrades within Community scope.
2. **Stay on MariaDB, need project depth:** **Expert Consulting and Services** for architecture, deployment, audits, performance tuning, and operational hardening across MariaDB versions and environments where Support scope does not apply.
3. **Move to Percona software for MySQL:** **Controlled migration** to Percona Server for MySQL (and related Percona tools: backup, HA, Kubernetes operators, PMM) with expert-led or customer-led execution. Compatibility varies by workload, version, and features in use; validate in your environment.
4. **Operational ownership without a DBA bench:** **ExpertOps** for proactive database management when the customer wants Percona to run defined operational workflows alongside Support.

### Why this offering exists

- Percona is committed to open source and to helping teams avoid proprietary lock-in. We support technologies customers can inspect, operate on their own terms, and scale without license-driven limits.
- Percona has delivered MySQL software, support, ExpertOps, and consulting for more than 16 years, with enterprise-grade capabilities in **100% open source** Percona Server for MySQL and related tooling.
- From MariaDB 10.0 onward, MariaDB and MySQL diverged; MariaDB is not a universal drop-in for equivalent MySQL releases. Percona does **not** provide full support for **MariaDB Enterprise** software.
- Percona Server for MySQL is fully open source, with enterprise security and operational features, and production use without proprietary database licensing for the Percona distribution.

### Customer Challenges and Value Alignment – MariaDB

**Optimized TCO**

- **Open source path off proprietary MariaDB licensing:** Percona software for MySQL runs on-premises, in the cloud, and in hybrid environments. Teams can scale without MariaDB Enterprise-style license escalation when migration is the right fit.
- **Stay on Community without guessing scope:** Expert Support for MariaDB Community gives predictable expert access for production issues without implying coverage of Enterprise editions.

**Performance and Reliability at Scale**

- **Keep MariaDB stable in production:** Expert Support and consulting help resolve performance regressions, replication and HA issues, backup and recovery gaps, and upgrade risk.
- **Kubernetes and cloud-neutral operations:** When migration targets Percona's MySQL stack, Percona Kubernetes Operators for MySQL provide backup/restore, HA, replication, and logging patterns without tying operations to a single vendor DBaaS.

**Security, Sovereignty, and Compliance**

- **Community support with expert validation:** Percona engineers help customers maintain operational control while meeting internal security and compliance requirements for supported MariaDB Community deployments.
- **Migration option with inspectable software:** Moving to Percona Server for MySQL keeps the database layer open source and customer-operated, with transparent components for security review.

**Adaptability for Emerging Workloads**

- **Database-agnostic expertise:** Consulting draws on in-house MySQL, MariaDB, PostgreSQL, and MongoDB specialists for architecture, migration, and performance work.
- **Training:** Percona offers [training](https://www.percona.com/training) on MariaDB and MySQL tailored to the customer's use case.
